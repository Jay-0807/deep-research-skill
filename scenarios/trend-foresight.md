# 场景：趋势 / 前瞻调研 (Trend & Foresight)

## 何时触发
用户问："X 的趋势"、"X 的未来"、"X 还能火多久"、"下一波 Y 会是什么"、"X 的早期信号"、"X 5 年后什么样"、"X 是不是泡沫"、"X 的拐点信号"。

## 必答 5 大问题

1. **当前趋势**：3-5 个塑造该领域的活跃趋势 + 信号源
2. **弱信号扫描**：还未主流化、但有 2-3 个独立信号的早期 pattern
3. **驱动因素**：技术 / 监管 / 消费者 / 资本 / 地缘 哪些在推 / 哪些在拖
4. **三情景预测**：3-5 年后的 Bull / Base / Bear 情景
5. **拐点信号**：什么信号出现表明进入下一阶段（用作监控指标）

## 推荐方法论

### 1. STEEP 框架（必做的 5 维扫描）
对任何趋势话题，先扫 5 个维度的驱动力：
- **Social** 社会：人口 / 价值观 / 生活方式 / 阶层流动
- **Technological** 技术：新技术成熟度 / 关键 enabler / 主流采纳曲线
- **Economic** 经济：成本结构变化 / 资本流向 / 商业模式可行性
- **Environmental** 环境：气候 / 能源 / 资源 / 监管驱动的绿色化
- **Political** 政治 / 监管：政府意志 / 国际关系 / 法规演化

每个维度至少 1-2 个具体信号。

### 2. 弱信号识别（Weak Signal Detection）

什么是"弱信号"：还未成为新闻头条，但已有 2-3 个独立来源指向同一方向。

**捕获弱信号的来源**：
- 顶会论文里的"未来工作" / 综述里的"open challenge"
- 早期 startup 的融资 / pivot（看资本嗅觉）
- 招聘平台新出现的岗位（看公司在赌什么）
- 监管草案 / 试点项目（看政策意图）
- Twitter / 即刻 / Substack 上行业 KOL 的"业余实验"
- 海外早期试点 / 区域性案例（看跨地区迁移可能）
- 小众 subreddit / 论坛的"反常增长"
- 专利申请的新方向（看 R&D 投入趋势）

**弱信号 ≠ 噱头**。判断标准：
- ≥3 个独立来源
- 跨越至少 2 个 STEEP 维度
- 有具体的 "what changed" 而不是"大家都说会有 X"

### 3. Crossing-the-Chasm（Gartner Hype Cycle 类）
对每个趋势，定位它在采纳曲线的哪个阶段：
- **Innovation trigger**：技术刚出现，少数人尝试
- **Peak of inflated expectations**：媒体狂热，估值飙升
- **Trough of disillusionment**：第一波失败，热度退潮
- **Slope of enlightenment**：实际可用的 use case 浮现
- **Plateau of productivity**：主流采纳 / 商业可持续

不同阶段对应不同投资 / 产品策略。

### 4. 三情景预测（5 年视野）

| 情景 | 关键假设 | 触发条件 | 主要受益者 | 主要受害者 | 概率 |
|------|---------|---------|----------|----------|------|
| Bull | 加速发生 | 列 2-3 条 | 列 | 列 | ~25% |
| Base | 渐进演化 | 列 2-3 条 | 列 | 列 | ~50% |
| Bear | 失败 / 倒退 | 列 2-3 条 | 列 | 列 | ~25% |

### 5. 拐点监控指标（Tripwire Metrics）
报告交付后用户能持续 monitor 的指标。每个趋势 3-5 个：
- **领先指标**：在事情发生前会变化（如 venture funding / 招聘数 / 专利申请）
- **同步指标**：与事情同时变化（如用户数 / 营收 / 媒体声量）
- **滞后指标**：事情发生后才变化（如行业报告大数 / 监管出台）

报告必须告诉用户："如果你看到指标 X 突破 Y 阈值，说明进入了 Base 情景；如果指标 Z 下行，准备进入 Bear。"

### 6. Delphi 法（轻量版）
对争议性强的趋势，找 ≥3 个独立专家 / KOL 的公开观点（他们的演讲、访谈、推文）并整理：
- 共识点
- 分歧点 + 各自论据
- 极端观点（用来 stress-test 自己的判断）

### 7. 类比 / 类推（Analogy reasoning）
"现在的 X 像不像过去的 Y？" 是有用但容易被滥用的方法。
- 必须列出至少 3 个相似维度 + 至少 1 个本质不同的维度（防过度类比）
- 类比对象越远（跨行业 / 跨时代），越要 surface 不同点

## 推荐信源

### 弱信号侦察
- **学术 / 专利**：参考 `sources/academic.md` + CNIPA / USPTO
- **早期资本**：Crunchbase 早期轮 / a16z / Sequoia 公开 thesis / Y Combinator batch
- **公司动作**：招聘趋势（拉勾 / LinkedIn）/ 产品功能更新 / 财报 forward guidance
- **政策**：国家发改委 / 工信部 / 国务院发文 / 各部委试点公告 / 国务院学位委员会的专业目录变化
- **KOL**：Twitter / 即刻 / 小宇宙播客 / Substack / 微信公众号

### 主流信号验证
- `sources/industry.md` 行业报告
- `sources/general-web.md` 媒体新闻

### 海外趋势先发地
- 美国：The Information / Stratechery / Lenny's Newsletter
- 日本：日经 / Diamond Online
- 欧洲：FT
- 这些地区出现的现象，1-3 年常传到中国

## 推荐模板
`templates/report.md` + 额外的"趋势卡片"小模板（每个趋势用同一格式）

### 趋势卡片格式
```markdown
### 趋势 N：[名称]
- **What's happening**：一句话描述
- **Why now**：为什么是现在 / 触发因素
- **Signals**（3 个独立信号）
  - 信号 1：[具体事件 / 数据] - [源]
  - 信号 2：...
  - 信号 3：...
- **Hype cycle 位置**：[阶段]
- **STEEP 驱动维度**：[S/T/E/E/P 哪些在推]
- **5 年 Base case**：...
- **Tripwire**（监控指标）：...
- **反方观点**：...
```

## 输出 deliverables
1. **TL;DR**（最大趋势 + 最被忽视的弱信号）
2. **当前主流趋势**（3-5 个，每个用趋势卡片格式）
3. **弱信号扫描**（2-4 个早期 pattern）
4. **STEEP 驱动分析**
5. **三情景 5 年预测**
6. **拐点监控指标**
7. **References**

## 反模式
- 把"趋势"写成"已经发生的事"（必须有前瞻性，不只是新闻总结）
- 信号 <3 个就声称看到趋势
- 没有反方 / 失败情景
- 类比过度（任何 X like Y 都要带"但 X 和 Y 在 Z 维度本质不同"）
- 把 hype 当趋势（必须 surface "我担心这只是 hype" 的可能性）
- 没有 monitoring tripwire（趋势报告不可监控就没价值）
