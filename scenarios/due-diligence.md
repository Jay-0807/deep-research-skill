# 场景：公司 / 投资尽调 (Due Diligence)

## 何时触发
用户问："调研 X 公司"、"X 公司值不值得投"、"X 公司的背景"、"X 公司的财务"、"X 公司有什么风险"、"对 X 做尽调"。

## 必答 8 大问题

1. **公司基本面**：成立时间、注册地、股权结构、核心团队、员工规模
2. **业务模式**：产品 / 收入来源 / 客户类型 / 增长引擎
3. **市场地位**：行业排名、市场份额、关键客户
4. **财务表现**：收入 / 利润 / 现金流 / 毛利 / 增长率（如可获得）
5. **融资历史**：每轮估值 / 投资方 / 用途
6. **核心风险（红旗扫描）**：法律 / 财务 / 团队 / 监管 / 客户集中度
7. **三情景预测**：Bull / Base / Bear case
8. **决策建议**：投 / 不投 + position sizing + 监控指标

## 推荐方法论

### 1. Bull / Base / Bear 三情景（必做，来自 quant-equity-research）
对每家公司构建三情景：

| 情景 | 假设 | 关键驱动 | 预期回报 / 12 个月路径 | 概率 |
|------|------|---------|--------------------|------|
| Bull | 一切顺利（市场快增、关键合作落地、产品 PMF 加深） | 列 3 个 | 上行空间 | ~25% |
| Base | 大概率路径（增速温和、竞争加剧） | 列 3 个 | 中位预期 | ~50% |
| Bear | 主要风险兑现（关键客户流失、监管收紧、技术替代） | 列 3 个 | 下行风险 | ~25% |

最终估值 = 概率加权。

### 2. 红旗扫描（Red Flag Checklist）
9 项必查：
- ☐ 法律：有无未结诉讼 / 行政处罚 / 知识产权纠纷
- ☐ 股权：股权结构是否清晰 / 是否有代持 / VIE 风险（中国公司）
- ☐ 团队：创始人/CEO 履历是否完整 / 关键高管离职率 / 团队内斗信号
- ☐ 财务：审计意见是否标准无保留 / 关联交易是否合理 / 现金流是否健康
- ☐ 客户：客户集中度（Top 5 收入占比）/ 关键客户合同到期
- ☐ 监管：所在行业政策风险 / 数据合规 / 行业准入
- ☐ 技术：核心技术来源 / 专利情况 / 技术债 / 安全事故
- ☐ 媒体：负面新闻 / 创始人/高管个人争议 / 网络口碑趋势
- ☐ 供应商：供应链集中度 / 关键供应商稳定性

每项打分：[Green / Yellow / Red]。**任意 Red 必须在报告 Exec Summary 显式提及**。

### 3. 三轴 cross-reference（防营销稿污染）
任何 claim 都要从三个角度交叉印证：
- **公司自己说**（官网、PR、招股书、CEO 访谈）
- **第三方报道**（媒体、行业报告、分析师）
- **客观数据**（财报、SEC 文件、企查查工商、企业信用公示）

三处一致 → 高可信；任何一处不一致 → red flag 候选。

### 4. 团队背调（重要常被忽略）
- 创始人：LinkedIn 履历 / 是否完整 / 过往公司是否成功
- 高管：来自哪里 / 工龄稳定性 / 是否有重复跳槽
- 关键技术负责人：是否有 GitHub / 论文 / 业界声誉
- **离职率信号**：脉脉 / Glassdoor / Blind 上看在职员工评价
- 中国公司额外查：股权关系 / 关联企业 / 历史变更

## 推荐信源

### 中国公司
- **必查**：企查查 / 天眼查 / 启信宝 / 国家企业信用信息公示系统
- 上市公司：巨潮资讯（cninfo.com.cn）/ 港交所披露易 / SEC EDGAR（中概股）
- 监管处罚：信用中国（creditchina.gov.cn）/ 各部委处罚公告
- 招聘信号：拉勾 / Boss 直聘的招聘趋势
- 团队评价：脉脉 / 知乎 / V2EX
- 媒体：财新 / 36 氪 / 晚点 / 36 氪深度 / 雷锋网

### 海外公司
- **必查**：SEC EDGAR（US 上市） / Companies House（UK） / OpenCorporates
- Crunchbase / PitchBook / Tracxn（融资）
- LinkedIn / The Org（团队结构）
- Glassdoor / Blind（员工评价）
- 媒体：FT / WSJ / Bloomberg / Reuters / The Information

## 输出 deliverables
1. **TL;DR 投资判断**（一句话：投 / 观望 / 否决）
2. **公司概况**（基本面表）
3. **业务模式拆解**（产品 / 客户 / 收入）
4. **市场地位 + 竞争优势**
5. **财务表现**（如可获得）
6. **三情景估值 / 预测**
7. **红旗扫描结果**（9 项 + Green/Yellow/Red）
8. **团队评估**
9. **关键监控指标**（投后跟踪用）
10. **References**

## 反模式
- 只看公司官网和 PR → 必有 ≥3 个第三方源交叉
- 没扫红旗 → 必须 9 项全扫
- 财务数据用厂商自报 → 必须找审计版本（财报、招股书）
- "投资人很看好" → 必须列具体投资人 + 估值变化曲线
- 没有 Bear case → 必须有，且要 take seriously
- 把"创始人是行业老兵"当作判断 → 必须验证具体经历的可验证细节
