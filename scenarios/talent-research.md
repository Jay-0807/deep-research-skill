# 场景：人才 / 招聘调研 (Talent / Hiring Research)

## 何时触发
用户问："调研 X 候选人"、"X 岗位的薪资水平"、"X 领域的人才地图"、"X 公司挖人"、"哪些公司的 X 团队最强"、"X 岗位现在招的人是什么样"。

> 这是生态空白点 —— 现有 skill 都是企业内部 HR / 面试设计，没人做"候选人背调 / 人才地图 / 薪资 benchmark"。本场景填补。

## 必答 5 大问题（根据 query 选 3-5）

1. **候选人背调**：教育 / 工作经历 / 项目成果 / 业界声誉 / 红旗信号
2. **薪资 benchmark**：岗位在不同城市 / 公司规模 / 行业的薪资分布 + 涨幅趋势
3. **人才地图**：某领域 Top N 人才的所在公司 + 流动趋势 + 可挖性
4. **团队画像**：某公司某团队的规模 / 资历分布 / 技能结构 / 离职率
5. **招聘趋势**：岗位需求量变化 / 技能要求演化 / 雇主品牌竞争

## 不同子任务的方法论

### A. 候选人背调（Background Check）

**5 维度核查 + 红旗扫描**：
1. **教育**：学校 / 专业 / 毕业年 → 学信网 / LinkedIn / Google 学术（如有）
2. **工作经历**：每段任职时间 / title 进阶 / 公司变更 → LinkedIn / 脉脉 + Google 二次核实
3. **项目成果**：GitHub / 论文 / 专利 / 公开演讲 / 媒体提及
4. **业界声誉**：圈内人对 ta 的评价（脉脉 / Blind / Glassdoor）
5. **红旗信号**：
   - 工作时间 gap（无解释的空窗 >6 个月）
   - 频繁跳槽（<1 年的多次跳）
   - 简历夸大（title vs 实际职责对不上）
   - 公开争议 / 诉讼
   - 离职原因模糊
   - 同行评价两极

**伦理边界**：
- ⚠️ 候选人背调必须告知候选人或在合法授权范围（如 reference check）
- ⚠️ 不主动挖隐私（社交媒体私人生活、住址、家庭、医疗）
- ⚠️ 不爬已删除内容 / 不绕过权限设置
- 本 skill 严格限制在 **公开可访问的职业信息**

### B. 薪资 Benchmark

**3 个维度交叉**：
- 城市（北上深杭 / 二线 / 海外）
- 公司类型（大厂 / 中厂 / 创业 / 外企）
- 资历 / level（应届 / 3 年 / 5 年 / 10 年 / 总监级）

**3 种数据源交叉验证**：
1. **招聘平台公开 JD**（拉勾 / Boss / LinkedIn 的薪资范围）
2. **薪资爆料平台**（脉脉 / Blind / Levels.fyi / OfferShow）
3. **行业薪酬报告**（Robert Half / Hudson / 任仕达 / 猎聘）

**报告内容**：
- P25 / P50 / P75 / P90 分位数（基本薪 + total comp）
- 同比变化（涨 / 平 / 跌）
- 异常点（远高于 / 远低于市场的 case 解释）

### C. 人才地图（Talent Map）

**针对某领域（如"AI Agent 工程师"/"半导体光刻工程师"）找 Top 50-100 人才**：

1. **识别**：
   - 顶会论文一作 / 知名开源 maintainer / 高引用研究员 / 著名公司技术负责人
   - 行业 KOL（公众号 / Twitter / 知乎 / 播客主理人）
   - 招聘平台搜 senior+ title

2. **属性提取**：
   - 当前公司 + 职位
   - 工龄
   - 标志性成果
   - 公开活跃度（是否经常分享）
   - 上一份 / 上上份公司（流动路径）

3. **可挖性评估**：
   - 在当前公司年限（>5 年通常稳定 / 1-3 年高流动期）
   - 是否处于"成果爆发期"（积极发声）vs "沉淀期"（潜水）
   - 公开抱怨当前雇主的信号
   - 公开兴趣点（如果与目标公司方向一致，可挖性高）

### D. 团队画像

针对某公司的某团队（如"字节豆包大模型团队"）：
- LinkedIn / 脉脉 搜公司 + title 过滤 → 估算团队规模
- 抽样 20-50 人看简历分布 → 资历结构 / 来源公司 / 工龄
- 看团队 leader 的公开演讲 / 文章 → 团队技术文化
- 找近 1-2 年加入 / 离开的人 → 增长 vs 流失

### E. 招聘趋势

- 同一职位过去 12-24 个月的 JD 数量趋势（招聘平台搜索量）
- JD 的关键技能要求演化（X 技能从可选变必选 / 新增 Y 技能要求）
- 雇主品牌竞争（哪些公司在抢这个岗位）
- 薪资变化（参考 B）

## 推荐信源

### 中文
- **职业信息**：LinkedIn 中国 / 脉脉 / Boss 直聘 / 拉勾 / 智联 / 猎聘
- **薪资爆料**：脉脉职言 / OfferShow / 看准网
- **学历核实**：学信网（仅本人或授权可查）
- **公开成果**：GitHub / 知乎专栏 / 公众号 / 即刻 / 极客时间专栏
- **行业讨论**：脉脉 / V2EX / 即刻 / 微博职场话题

### 英文
- **职业信息**：LinkedIn / The Org / Crunchbase 团队页 / 公司 about us
- **薪资爆料**：Levels.fyi / Blind / Glassdoor / Comparably
- **公开成果**：GitHub / Google Scholar / Twitter / Medium / personal blog
- **行业讨论**：HackerNews / Reddit (r/cscareerquestions 等) / Blind

## 推荐模板
`templates/report.md` 通用模板，按子任务定制各 section

## 输出 deliverables

按子任务不同：

**候选人背调**：
1. 候选人快照（基本信息表）
2. 5 维度核查结果
3. 红旗扫描（标 Green/Yellow/Red）
4. 综合判断 + 推荐 reference check 联系人方向

**薪资 benchmark**：
1. 三轴分位数表
2. 同比趋势
3. 异常点解释
4. 建议薪资 offer 范围

**人才地图**：
1. Top N 人才表（公司 / 职位 / 工龄 / 成果 / 可挖性）
2. 流动路径图
3. 重点人选推荐

**团队画像**：
1. 团队基本面（规模 / 资历 / 来源）
2. 技术文化判断
3. 近期人员变动信号

**招聘趋势**：
1. JD 数量曲线
2. 技能要求演化表
3. 雇主竞争格局
4. 薪资变化

## 反模式 + 红线

- ❌ 爬私人社交账号（朋友圈 / 私人 IG）
- ❌ 调查家庭 / 住址 / 健康
- ❌ 联系候选人的现在同事打探（除非通过正式 reference check 流程）
- ❌ 使用没授权的内部薪资数据
- ❌ 仅凭一份脉脉发言判断候选人（必须三源交叉）
- ❌ 把候选人个人观点 / 政治倾向作为评估依据
- 必须区分 **职业能力评估**（OK）和 **个人调查**（NOT OK）
