# 场景：竞品调研 (Competitive Intelligence)

## 何时触发
用户问："X 的竞品有哪些"、"对比 A 和 B"、"调研 X 赛道的玩家"、"SWOT 分析 X"、"X 的定位"、"benchmark"、"竞争格局"。

## 必答 6 大问题

1. **玩家清单**：这个赛道有哪些公司？分几个梯队？
2. **定位地图**：每个玩家的差异化坐标（横纵轴二维定位）
3. **功能对比**：产品功能矩阵（feature × player）
4. **商业模式**：定价、客户、收入结构、增长引擎
5. **优劣势 SWOT**：每个 Top 玩家的 SWOT
6. **威胁与机会**：哪些是结构性威胁、哪些是潜在机会

## 推荐方法论

### 1. STERN（先扫已有报告）
按 `methodology/pipeline.md` 的 Phase 2 策略 A：先找 a16z / Sequoia / Gartner / 艾瑞 / 36 氪 已有的 landscape 报告，反推 outline。

### 2. 玩家筛选规则
- 不要列 >15 个玩家。把"不会真正竞争"的剔除（地理隔离、客群完全不重叠、产品形态根本不同）
- 用 ≥3 个独立信源印证才纳入清单
- 分梯队：Tier 1（市场主导）/ Tier 2（增长黑马）/ Tier 3（垂直/区域）/ Tier 4（早期/潜在颠覆者）

### 3. 定位地图的两轴选择
- 最常见组合：「价格 × 功能完整度」、「自动化 × 定制化」、「企业 × 个人」、「广度 × 深度」
- **必须在 Phase 1 澄清时让用户确认两轴**，否则后续工作会跑偏

### 4. fatal-flaw hypothesis（来自 VoltAgent/project-idea-validator）
对每个 Top 3 竞品，主动假设"这家公司有什么致命弱点"。从需求侧（用户不够痛）、供给侧（差异化薄弱）、采纳侧（switching cost 高）三个角度找证据。这种 adversarial 视角能避免被 marketing 内容洗脑。

## 推荐信源
- 主要：`sources/general-web.md`（找公司公开信息）+ `sources/company.md`（融资/团队/财务）
- 中文场景必加：IT 桔子、36 氪、亿欧、企查查
- 用户反馈：G2、Capterra、Product Hunt、知乎专业问答、小红书种草/吐槽

## 推荐模板
`templates/competitive-matrix.md`（含对比表 + 定位地图代码块）

## 输出 deliverables
报告里必须包含：
1. 玩家清单表（公司 / 创立年 / 融资 / 员工 / 主要市场 / 核心产品）
2. 功能矩阵表（必须）
3. 定位地图（ASCII 图或 markdown 表格表示坐标）
4. 每个 Top 3 玩家 1 段 SWOT
5. 威胁矩阵（按"可能性 × 影响"打分）
6. 决策建议（如果用户是产品/创业者：应该卡哪个差异化位；如果是投资人：哪家更值得投）

## 反模式
- 列 20+ 玩家堆字数 → 砍到 10 以内
- 只看 marketing 页面 → 必须看用户实际评价
- 把竞品报告变成新闻稿堆叠 → 必须有"我的判断"
- 没有定位地图 → 必须有
- 没有威胁/机会 → 必须有
